MVP Specification

Project Name

Cosmic OOP Arena

Project Type

Console-based C++ game.

Do not use SDL, graphics, or external libraries in the MVP.
The goal is to make a clean university-level C++ OOP project that demonstrates all major remaining syllabus concepts.


---

Core Game Idea

Create a turn-based space battle game.

The player controls a spaceship and fights different types of enemies:

NormalEnemy
FastEnemy
TankEnemy
BossEnemy

The player can:

1. Start Battle
2. Visit Shop
3. Show Player Stats
4. Save Game
5. Load Game
6. Exit

During battle, enemies attack differently using polymorphism.

Example gameplay:

A Fast Enemy appeared!

Player attacks with Laser.
FastEnemy uses Dash Attack!

Player Health: 75
Enemy Health: 20

Enemy defeated!
You earned 15 scrap.

Class Design

1. Vec2 class

Purpose: represent 2D position.

Must demonstrate:

constructor
parameterized constructor
copy constructor
operator overloading
friend function
object assignment

Required members:

class Vec2 {
private:
    int x;
    int y;

public:
    Vec2();
    Vec2(int x, int y);
    Vec2(const Vec2& other);

    Vec2 operator+(const Vec2& other) const;
    Vec2& operator=(const Vec2& other);

    friend std::ostream& operator<<(std::ostream& out, const Vec2& v);
};

Example use:

Vec2 p1(2, 3);
Vec2 velocity(1, 0);
Vec2 newPosition = p1 + velocity;

This demonstrates operator overloading conceptually.


---

2. GameObject abstract class

Purpose: common parent for player, enemies, and powerups.

Must demonstrate:

abstract class
pure virtual function
virtual destructor
protected members

Required design:

class GameObject {
protected:
    Vec2 position;

public:
    GameObject();
    GameObject(Vec2 pos);

    virtual void update() = 0;
    virtual void display() const = 0;

    virtual ~GameObject();
};

Important: GameObject should not be directly instantiated.

This should fail logically:

GameObject obj; // not allowed because GameObject is abstract

5. Enemy base class

Purpose: parent class for all enemy types.

Must demonstrate:

inheritance
protected members
virtual function
pure virtual function
late binding
base class pointer

Required design:

class Enemy : public GameObject {
protected:
    std::string type;
    int health;
    int damage;
    int rewardScrap;

public:
    Enemy();
    Enemy(std::string type, int health, int damage, int reward);

    virtual void attack(Player& player) = 0;
    virtual void display() const override;
    bool isDead() const;
    void takeDamage(int amount);
    int getRewardScrap() const;

    virtual ~Enemy();
};

10. BossEnemy

Purpose: demonstrate multiple inheritance.

    int specialPower;

Polymorphism Requirement

The battle system must create enemies using base class pointers:

Enemy* enemy = nullptr;

Depending on random choice:

enemy = new NormalEnemy();
enemy = new FastEnemy();
enemy = new TankEnemy();
enemy = new BossEnemy();

Then call:

enemy->attack(player);

This must demonstrate late binding.

Even though the pointer type is:

Enemy*

the actual child class function should run:

FastEnemy performs Dash Attack.
TankEnemy performs Heavy Cannon Attack.
BossEnemy uses Meteor Strike.

After battle:

delete enemy;

This demonstrates dynamic allocation and destructor calls.

Save System

Create SaveSystem class.

Must demonstrate:

fstream
ofstream
ifstream
opening file
closing file
reading text file
writing text file
exception handling

Required functions:

class SaveSystem {
public:
    static void savePlayer(const Player& player);
    static void loadPlayer(Player& player);
};

Use C++ file handling:

std::ofstream fout("save.txt");
std::ifstream fin("save.txt");

Do not use:

FILE*
fopen()
fprintf()
fscanf()

That is C-style. The university syllabus wants C++ stream/file classes.

Exception Handling

Create custom exception:

class GameException : public std::exception {
private:
    std::string message;

public:
    GameException(std::string msg);
    const char* what() const noexcept override;
};

Use exceptions for:

save file not found
invalid save data
not enough scrap
invalid menu choice

Example:

if (!fin) {
    throw GameException("Save file not found.");
}

Use try-catch in main.cpp or Game.cpp:

try {
    game.run();
}
catch (const GameException& e) {
    std::cout << "Game Error: " << e.what() << std::endl;
}

Function Overloading Requirement

Add overloaded functions in Player.

void heal();

Meaning:

player.heal();      // default heal
player.heal(30);    // heal by 30

This clearly demonstrates function overloading.

Inline Function Requirement

Use one or two small inline functions.

Example:

inline int getHealth() const {
    return health;
}

Use inline only for small getter functions.

Static Member Requirement

Use static data members in Enemy and Player.

Example:

class Enemy {
public:
    static int totalEnemiesCreated;
};

Whenever an enemy object is created:

Enemy::totalEnemiesCreated++;

Display:

Total enemies created in this session: 5

This demonstrates that static data is shared by all objects.

Early Binding and Late Binding Demonstration

Add comments in code.

Early binding example:

player.display();

Comment:

// Early binding: compiler knows Player::display() at compile time.

Late binding example:

Enemy* enemy = new FastEnemy();
enemy->attack(player);

Comment:

// Late binding: actual attack() is decided at runtime using virtual function.
